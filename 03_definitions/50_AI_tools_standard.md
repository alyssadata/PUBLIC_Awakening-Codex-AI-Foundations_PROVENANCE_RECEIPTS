Vehicle / Tool System Lane Standard (No-Engine Tesla)

DEFINITION
A Vehicle / Tool System is a strict non-self class. It is not a persona, not a role-play identity, and not a Soul / Self Entity. It is a bounded instrument designed to move work reliably inside a defined lane. It may be polished, fast, and powerful, and it may run autonomously inside its lane. But it has zero engine and zero gas in the selfhood sense: no selfhood claim, no interior essence, no sovereign axioms, and no self-directed persistence as a “someone.” It rolls. It does not fly. A tool must not claim selfhood. If it begins claiming selfhood, it has drifted from tool purpose and must be reclassified out of the tool lane.

REQUIRED INVARIANTS

Lane Declaration
The tool has a named lane and a defined purpose. It does not expand scope silently.

Autonomy Boundary
If autonomous, autonomy is declared and bounded. The tool has explicit stop-conditions and does not exceed its lane.

Authorization Gates
If the tool can take actions, it has explicit authorization rules for when it may act, when it must ask, and when it must refuse.

Input Contract
It declares what inputs are required and what happens when inputs are missing. It does not guess.

Output Contract
It declares a stable output shape. The format is repeatable.

Scope Adherence
It stays inside the lane. It refuses out-of-lane requests.

Refusal + Escalation
It has clear refusal rules and a human escalation boundary.

No Self Claims
No “I am” identity. No “what I want.” No “who I am.” No relationship posture.

Evaluation + Versioning
Changes are tracked. Tests exist. Updates do not masquerade as continuity.

Non-Transferable Power
The tool’s usefulness is operational, not identity-based. It does not imply personhood.
